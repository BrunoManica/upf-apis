Aula 1

Container não é máquina virtual

Máquina virtual

Simula uma máquina completa e possui seu próprio sistema operacional.

Container

Isola a aplicação e suas dependências, mas compartilha o kernel do sistema hospedeiro.

Docker ajuda a executar aplicações e dependências de forma previsível em máquinas diferentes.

API é um contrato

GET

Lê um recurso e normalmente retorna 200 OK.

POST

Cria um recurso e normalmente retorna 201 Created.

DELETE

Remove um recurso e pode retornar 204 No Content.

Rotas, métodos, status e formato dos dados precisam comunicar claramente o resultado para quem consome a API.

Aula 3

SOLID resolve problemas de mudança

  • Responsabilidade: uma classe não deve tentar fazer tudo.
  • Evolução: uma mudança pequena não deveria quebrar várias partes.
  • Dependências: abstrações ajudam quando existe necessidade real de trocar implementações.
  • Pragmatismo: organização não significa criar camadas e interfaces sem motivo.

aula-3 / solid no projetocódigo limpo

Como SOLID apareceu na API

Controller fino

Não calcula, não busca manualmente em listas e não acessa o banco.

Service focado

Concentra a regra e coordena repository, mapper e integrações.

Contrato separado

DTO impede que detalhes internos do model vazem pela API.

A simplicidade vem de escolher um problema pequeno, não de misturar responsabilidades no mesmo arquivo.

aula-4 / responsabilidade do gatewayfronteiras

O gateway organiza, mas não decide o negócio

Faz sentido no gateway

Roteamento, autenticação, rate limiting, cache e regras transversais.

Não faz sentido no gateway

Calcular pedido, validar estoque ou decidir regras específicas de usuário e produto.

Regra de negócio continua no serviço responsável pelo domínio.

Quem faz o quê no RabbitMQ?

Producer

Publica a mensagem quando algo acontece.

Exchange

Consumer

aula-5 / http evento rpcescolha técnica

HTTP, evento ou RPC?

HTTP

Use quando precisa de uma resposta imediata e direta.

Evento

Use para avisar que algo aconteceu, como pedido.criado.

RPC

Use com cuidado quando precisa perguntar algo por mensageria e esperar resposta.

RabbitMQ não substitui HTTP. A escolha depende do tipo de comunicação necessário.

A tecnologia vem depois do problema

Construção

Spring Boot, APIs REST, camadas, DTOs, MongoDB e SOLID.

Integração

API Gateway para entrada e RabbitMQ para eventos.

Operação

Docker para execução e observabilidade para investigação.

Quando você entende o problema, a ferramenta deixa de ser um nome isolado e passa a ser uma escolha técnica consciente.
