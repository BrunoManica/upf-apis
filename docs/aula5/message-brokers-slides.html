rabbitmq-caso-real.txt
_
[]
x

Caso iFood

Problema

O pedido foi criado, mas enviar notificacao, confirmar loja e gerar cobranca nao precisam travar a tela.

Solucao

Usar RabbitMQ com uma exchange de pedidos e filas separadas para notificacao, faturamento e analytics. A API publica pedido.criado uma vez; cada worker processa sua fila sem bloquear o cliente.

RabbitMQ ajuda quando existe trabalho claro para ser distribuido entre consumidores.

kafka-caso-real.txt
_
[]
x

Caso Uber

Problema

Muitas localizacoes, status de corrida e eventos chegando o tempo todo.

Solucao

Usar topicos do Kafka para receber eventos como corrida.iniciada, motorista.localizacao_atualizada e corrida.finalizada. Diferentes consumer groups leem o mesmo historico para mapa em tempo real, cobranca e analytics.

Nao e sobre uma tarefa unica. E sobre acompanhar um fluxo continuo de eventos.
