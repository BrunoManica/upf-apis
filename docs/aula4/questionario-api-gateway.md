# Questionário: API Gateway

## Perguntas fáceis

1. O que é um API Gateway em uma arquitetura com vários serviços?

2. Qual problema aparece quando o cliente precisa chamar diretamente vários microsserviços diferentes?

3. Com API Gateway, o cliente precisa conhecer o endereço interno de cada serviço? Explique.

4. Em uma frase, qual é a principal função do API Gateway?

## Perguntas médias

5. Explique a diferença entre chamar os serviços diretamente e chamar os serviços por meio de um API Gateway.

6. O que é roteamento dentro de um API Gateway? Use um exemplo com `/api/users` e `/api/products`.

7. O que são filtros em um API Gateway? Cite dois exemplos de filtros mencionados na aula.

## Perguntas difíceis

8. Por que não é uma boa ideia colocar regra de negócio dentro do API Gateway?

9. Explique a diferença entre autenticação e rate limiting no contexto de um API Gateway.

10. Por que cache no API Gateway pode melhorar o desempenho, mas também pode causar problemas se for usado no lugar errado?
